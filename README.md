# README – Comparação de Resultados entre Diferentes Inteligências Artificiais

## Objetivo do Projeto

Este projeto teve como objetivo aplicar a técnica de **Prompt Estruturado** para a criação de uma Landing Page de um Pet Shop de Alto Padrão especializado em Banho e Tosa, utilizando diferentes ferramentas de Inteligência Artificial.

A proposta consiste em utilizar o mesmo prompt em múltiplas IAs e comparar os resultados gerados, analisando aspectos como qualidade do código, organização, criatividade, usabilidade, design e aderência aos requisitos solicitados.

---

## Metodologia

Foi elaborado um prompt estruturado contendo informações sobre:

* Objetivo do site;
* Público-alvo;
* Estrutura da Landing Page;
* Regras de HTML;
* Regras de CSS;
* Regras de JavaScript;
* Requisitos de UX e segurança.

O mesmo prompt foi utilizado em diferentes plataformas de Inteligência Artificial para verificar como cada modelo interpreta e implementa as mesmas instruções.

---

## Requisitos Adicionados ao Prompt

Durante a atividade foram inseridos requisitos extras para enriquecer o resultado:

### Contexto (UX)

* Exibição do tempo estimado de cada serviço.
* Destaque visual com selo "Mais Vendido" no serviço principal.
* Navegação simples e intuitiva.
* Foco na experiência do usuário.

### CSS

* Utilização de fontes sem serifa modernas.
* Interface visual premium.
* Botões com efeito de pulsação utilizando Keyframes.
* Layout responsivo.

### JavaScript

* Bloqueio de datas anteriores à data atual.
* Máscara automática para telefone.
* Validação dos campos do formulário.
* Melhorias de usabilidade.

---

## Critérios de Comparação

Os resultados gerados pelas IAs foram analisados considerando:

### Estrutura do Código

Verificação da organização dos arquivos HTML, CSS e JavaScript.

### Design e Interface

Avaliação da aparência visual, responsividade e qualidade da experiência do usuário.

### Funcionalidades

Análise das funcionalidades solicitadas no prompt, incluindo:

* Formulário de agendamento;
* Validação de dados;
* Máscara de telefone;
* Controle de datas;
* Animações.

### Qualidade do Código

Observação da clareza, legibilidade e boas práticas de desenvolvimento.
